# AI 모델 학습을 위한 데이터 요구사항 및 개선안

## 현재 상황 분석

### ✅ 현재 수집 가능한 데이터
1. **시간별 경고 로그**: 어느 시간에 어떤 센서값이 부족한지
2. **시간별 복귀 로그**: 정상으로 돌아온 시간
3. **로그에서 파싱한 센서값**: 주의 로그 메시지에서 추출한 값 (불완전)

### ❌ 부족한 데이터
1. **연속적인 센서 시계열 데이터**: 정상 상태의 센서값도 포함한 전체 데이터
2. **실제 생산량 데이터**: 수확량, 생산량 등 결과 데이터
3. **작물 종류별 정보**: 작물별 최적 환경 조건
4. **환경 조치 이력**: 사용자가 취한 조치와 그 결과
5. **정상 상태 데이터**: 경고가 없는 정상 상태의 센서값

## 필요한 데이터 수집 개선안

### 1. 실시간 센서 데이터 수집 강화
**현재 문제**: 로그에서만 센서값을 추출하여 불완전함
**개선안**: 
- PLC에서 주기적으로(예: 1분마다) 센서값을 수집하여 저장
- 정상 상태든 경고 상태든 모든 데이터 수집
- 시계열 데이터베이스 또는 파일에 저장

### 2. 생산량 데이터 수집
**필요한 정보**:
- 수확일
- 수확량 (kg, 개 등)
- 작물 종류
- 재배 기간

**수집 방법**:
- UI에 "수확량 입력" 기능 추가
- 또는 자동 수집 (PLC에서 생산량 센서가 있다면)

### 3. 작물별 최적값 정보
**필요한 정보**:
- 작물 종류별 최적 온도 범위
- 작물 종류별 최적 습도 범위
- 작물 종류별 최적 채광 범위
- 작물 종류별 최적 토양습도 범위

**수집 방법**:
- FarmManageForm에서 설정한 작물 정보 활용
- 작물별 최적값 데이터베이스 구축

### 4. 환경 조치 이력
**필요한 정보**:
- 조치 시간
- 조치 내용 (예: "온도 조절", "습도 증가", "채광 조절")
- 조치 전 센서값
- 조치 후 센서값
- 조치 효과 (경고 해소 여부, 복귀 시간)

**수집 방법**:
- 사용자가 UI에서 조치를 기록하도록 기능 추가
- 또는 자동 감지 (임계값 변경, 수동 조절 등)

### 5. 정상 상태 데이터 수집
**현재 문제**: 경고 로그에서만 데이터를 추출하여 정상 상태 데이터 부족
**개선안**:
- 주기적으로 정상 상태의 센서값도 수집
- 예: 5분마다 정상 상태 데이터 저장

## 모델 학습을 위한 최소 데이터 요구사항

### 평균 데이터 분석 모델
- **최소 필요**: 100개 이상의 센서값 샘플 (정상 + 경고)
- **권장**: 1000개 이상
- **데이터 구성**: 
  - 센서값 (습도, 온도, 채광, 토양습도)
  - 타임스탬프
  - 상태 (정상/경고/복귀)

### 이상 징후 감지 모델
- **최소 필요**: 
  - 정상 상태 데이터: 200개 이상
  - 이상 상태 데이터: 50개 이상
- **권장**: 
  - 정상 상태: 1000개 이상
  - 이상 상태: 200개 이상
- **데이터 구성**:
  - 센서값 시계열
  - 이상 발생 시간
  - 이상 유형 (낮음/높음)

### 생산량 예측 모델
- **최소 필요**: 
  - 센서 데이터와 생산량이 매칭된 데이터: 30개 이상
- **권장**: 100개 이상
- **데이터 구성**:
  - 재배 기간 동안의 평균 센서값
  - 경고 발생 횟수
  - 실제 생산량
  - 작물 종류

## 즉시 개선 가능한 사항

### 1. 정상 상태 센서값 수집 추가
- UpdateSensorsFromPlc()에서 센서값을 읽을 때마다 AI 모델에 전달
- 경고 상태와 관계없이 모든 데이터 수집

### 2. 센서값 저장 주기 설정
- 현재는 로그 기반 수집
- 주기적 저장 기능 추가 (예: 5분마다)

### 3. 생산량 입력 기능 추가
- UI에 생산량 입력 버튼 추가
- 수확 시 생산량과 함께 센서 데이터 저장

### 4. 작물 정보 활용
- FarmManageForm의 작물 정보를 AI 모델에 전달
- 작물별 최적값과 비교 분석

## 권장 구현 순서

1. **1단계 (즉시)**: 정상 상태 센서값 수집 추가
2. **2단계 (단기)**: 주기적 센서 데이터 저장 기능
3. **3단계 (중기)**: 생산량 입력 기능 추가
4. **4단계 (장기)**: 환경 조치 이력 수집 기능

## 결론

현재 로그에서 시간별로 어디 센서값이 부족한지만으로는 **AI 모델 학습이 충분하지 않습니다**.

**최소한 필요한 것**:
1. 정상 상태의 센서값도 수집
2. 연속적인 시계열 데이터
3. 생산량 데이터 (생산량 예측 모델을 위해서)

**권장 사항**:
- 실시간 센서 데이터를 주기적으로 저장
- 생산량 입력 기능 추가
- 작물별 최적값 정보 활용
